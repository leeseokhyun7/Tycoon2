# AGENTS.md

## 기본 목표
이 저장소는 모바일 웹 미니게임 모음입니다.
코드는 단순히 작동하는 수준이 아니라, 사용자가 바로 플레이할 수 있는 완성도까지 갖춰야 합니다.

## 공통 기준
- 모바일 우선으로 구현한다.
- 390px 너비 화면에서 버튼, UI, 텍스트가 겹치지 않아야 한다.
- 게임 시작, 진행, 실패, 성공, 다시하기 흐름이 있어야 한다.
- 조작 버튼은 엄지로 누르기 쉬운 위치에 둔다.
- 화면 중앙을 불필요한 텍스트나 버튼으로 가리지 않는다.
- 설명 텍스트는 짧게 하고, 플레이를 방해하지 않게 한다.
- CSS는 임시 느낌이 나지 않게 정리한다.
- 색상, 간격, 버튼 크기, 폰트 크기는 게임 분위기에 맞게 조정한다.

## 게임 완성 기준
작업이 끝나면 아래를 반드시 확인한다.

1. 브라우저 콘솔 오류 없음
2. 모바일 화면에서 UI 겹침 없음
3. 게임 시작 가능
4. 게임 진행 가능
5. 실패 또는 종료 조건 작동
6. 다시하기 가능
7. 수정한 파일 목록 요약
8. 검증한 항목과 검증하지 못한 항목 구분

## 작업 방식
- 기존 파일 구조를 먼저 파악한다.
- 불필요하게 파일을 많이 늘리지 않는다.
- 단순한 데모가 아니라 실제로 플레이 가능한 결과물을 만든다.
- 애매한 부분은 가장 자연스러운 게임 UX 기준으로 판단한다.
- 작업 후에는 직접 실행하거나 가능한 테스트 명령을 실행한다.



# 붕어빵 타이쿤 — 이미지 에셋 가이드

게임 파일과 같은 위치에 `assets` 폴더를 만들고 아래 파일명으로 이미지를 넣으면, 게임이 자동 감지해서 SVG 대신 사용합니다. 파일이 없으면 기존 SVG 그래픽으로 동작하므로 일부만 넣어도 됩니다.

## 필요한 파일 목록

| 파일명 | 용도 | 권장 크기 | 필수 조건 |
|---|---|---|---|
| `fish_raw.png` | 반죽 상태(굽기 전) 붕어빵 | 480×304 | 투명 배경, 3개 세트 |
| `fish_gold.png` | 노릇하게 익은 붕어빵 | 480×304 | 위와 동일 구도·크기 |
| `fish_burnt.png` | 탄 붕어빵 | 480×304 | 위와 동일 구도·크기 |
| `customer_1.png` ~ `customer_8.png` | 손님 캐릭터 (있는 만큼 사용) | 360×408 | 투명 배경, 정면 |
| `city_bg.png` | 거리 배경 (게임+타이틀 공용) | 960×460 | 하단이 지면 |

주의사항: 붕어빵 3종은 반드시 같은 구도·같은 크기로 만들어야 합니다(익는 과정이 세 이미지의 크로스페이드로 표현되기 때문). 캐릭터는 발밑 여백을 최소화하면 바닥에 잘 붙어 보입니다.

## AI 이미지 생성 프롬프트

미드저니, DALL-E, 스테이블 디퓨전 등에 사용할 수 있는 프롬프트입니다. 캐릭터·소품은 `--no background` 또는 "transparent background, isolated on white" 를 붙이고, 생성 후 배경 제거 도구(remove.bg 등)로 투명화하세요.

### 붕어빵 3종 (같은 시드/구도 유지 권장)

```
cute kawaii taiyaki fish-shaped pastry, facing right, waffle grid texture,
smiling face with big eye, thick brown outline, mobile game item asset,
pastel cartoon style, soft shading, glossy highlight, isolated on white background
```

- raw: 위 프롬프트 + `pale cream uncooked batter color`
- gold: 위 프롬프트 + `golden brown perfectly baked, warm glow`
- burnt: 위 프롬프트 + `dark charred burnt, small smoke wisps, sad face`

### 손님 캐릭터 (8종 각각 변형)

변형 아이디어: 판다 캐릭터, 주황 고양이, 분홍 돼지, 곰, 할아버지, 어린이, 목도리 두른 여성, 야구모자 소년 등. "chibi panda character in winter coat..." 처럼 주어만 바꾸면 됩니다.

## 인트로/설명 화면도 직접 만들 수 있어요 (선택)

"새로 시작"을 누르면 나오는 인트로(설명) 슬라이드를, 직접 만든 이미지로 통째로 바꿀 수 있습니다.

- `assets/intro_1.png`, `intro_2.png`, `intro_3.png` … 처럼 **1번부터 이어지는 번호**로 넣으면, 넣은 장수만큼 슬라이드가 됩니다(최대 8장). 중간 번호가 비면 거기서 끊겨요(예: 1,2,4만 넣으면 1,2만 사용).
- 하나도 없으면 기존 기본 인트로(3장)가 그대로 나옵니다.
- 넘김(다음/이전 버튼·좌우 스와이프·점 인디케이터)과 마지막 장의 "장사 시작", 우상단 "건너뛰기"는 그대로 동작합니다.

**권장 크기: 세로형 1080×1920 (9:16) PNG.** 화면에 잘림 없이 통째로 들어가도록 맞춰집니다(`object-fit: contain`). 즉 이미지 안에 글씨·그림을 다 넣어 한 장의 안내 페이지로 만들면 됩니다. 폰 화면 비율과 다르면 위아래(또는 좌우)에 여백이 생길 수 있으니, 배경까지 꽉 채우려면 9:16으로 만들어 주세요.

## 속(팥·슈크림) 단면 이미지도 교체할 수 있어요 (선택)

속을 넣으면 붕어빵 몸통에 크림 테두리의 "단면"이 나타납니다. 기본은 코드가 그린 질감(팥=붉은 알갱이, 슈크림=노란 바탕에 흰 점)인데, 아래 파일을 넣으면 그 이미지를 속으로 사용합니다. 크림 테두리(테)는 코드가 자동으로 그려주므로, **이미지는 속 재료 부분만** 채워 그리면 됩니다.

| 파일명 | 용도 | 권장 크기 | 조건 |
|---|---|---|---|
| `fill_red.png` | 팥 속 질감 | 200×200 | 정사각, 배경 없이 속 재료로 꽉 채우기 |
| `fill_cream.png` | 슈크림 속 질감 | 200×200 | 위와 동일 |
| `fill_choco.png` | 초코 속 질감 | 200×200 | 위와 동일 |
| `fill_pizza.png` | 피자 속 질감 | 200×200 | 위와 동일 |

동작 방식: 이미지는 타원(단면) 모양으로 자동으로 잘려 채워집니다(`background: cover`). 그래서 **정사각형 한 장에 재료 질감을 가득** 그리면 됩니다(테두리·여백 불필요, 투명 배경도 필요 없음). 이 속 이미지는 화덕의 붕어빵뿐 아니라 손님 주문·쟁반·재료 선택 버튼의 작은 아이콘에도 똑같이 적용됩니다.

참고로 이것과 헷갈리기 쉬운 다른 훅이 하나 더 있어요:

- `fill_<맛>.png` = **속(단면) 질감만**. 붕어빵 그림은 그대로 두고 단면 안쪽만 이 이미지로 채움. (지금 설명한 것)
- `fish_<맛>.png` = 주문·쟁반·버튼의 **작은 붕어빵 아이콘 통짜 그림**을 맛별로 교체. 권장 크기 240×152(붕어빵 가로 비율), 투명 배경. 화덕의 큰 붕어빵에는 영향 없음.

대부분은 `fill_red.png` / `fill_cream.png` 두 장만 넣어도 충분히 그럴싸해집니다.

## 소리도 직접 교체할 수 있어요 (선택)

붕어빵 굽는 소리와 효과음도 `assets` 폴더에 아래 파일을 넣으면 자동으로 교체됩니다. 파일이 없으면 기존 합성음으로 동작하므로 원하는 것만 넣으면 됩니다.

| 파일명 | 용도 | 비고 |
|---|---|---|
| `sizzle.mp3` | 붕어빵 굽는 지글지글 소리 | 자동 반복 재생(끊김 없는 루프 권장), 굽는 개수에 따라 볼륨 자동 조절 |
| `pour.mp3` | 반죽 붓는 소리 | 짧게(1초 내외) |
| `flip.mp3` | 틀 뒤집는 소리 | 짧게 |
| `collect.mp3` | 다 구운 붕어빵 꺼내는 소리 | 짧게 |
| `sell.mp3` | 손님에게 판매 성공 소리 | 짧게, 경쾌하게 |
| `burnt.mp3` | 붕어빵 태웠을 때 소리 | 짧게 |
| `bgm.mp3` | **배경음악** (직접 만든 곡) | 게임 내내 자동 반복 재생. 넣으면 기본 생성 음악 대신 사용 |

### 배경음악(bgm.mp3) 넣는 법

`assets/bgm.mp3` 파일을 넣으면 게임이 자동 감지해 **기본 배경음악 대신 이 곡을 반복 재생**합니다(없으면 기존 자동 생성 음악). 만드실 때 참고:

- 형식은 `.mp3`, 길이는 30초~2분 정도면 충분해요(자동 반복되니 시작·끝이 자연스럽게 이어지는 루프면 더 좋습니다).
- 볼륨은 0.5로 재생되니 원곡은 적당히 크게 만들어도 됩니다.
- 우측 상단 ♪ 버튼으로 끄면 배경음악도 함께 멈춥니다.
- 브라우저 정책상 화면을 한 번 터치/클릭한 뒤부터 재생돼요.

- 형식은 `.mp3` 로 맞춰주세요(브라우저 호환성이 가장 좋습니다).
- 무료 효과음은 freesound.org(CC0 필터), Kenney 오디오 팩, mixkit.co 등에서 받을 수 있습니다.
- 넣은 뒤 게임을 새로고침하면 적용됩니다. 소리는 화면을 한 번 터치/클릭한 뒤부터 재생돼요(브라우저 정책).

## 무료 에셋 대안

AI 생성 대신 무료 에셋을 쓰려면 Kenney(kenney.nl)나 itch.io의 CC0 게임 에셋에서 캐릭터/음식 팩을 받아 위 파일명으로 저장하면 됩니다.

## 적용 방법

1. `Bungeoppang Tycoon` 폴더 안에 `assets` 폴더 생성
2. 위 파일명으로 PNG 저장
3. 게임 새로고침 — 자동 적용

이미지를 준비해서 폴더에 넣어주시면, 크기 조정·위치 보정·연출 개선은 제가 이어서 다듬어 드릴 수 있습니다.
